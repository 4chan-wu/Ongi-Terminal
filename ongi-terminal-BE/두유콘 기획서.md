# 📄 프로젝트 기획안: 온기 터미널 (Ongi-Terminal)

**슬로건:** “버리기 전에, 동네 안에서 다시 순환되도록”

## 1. 프로젝트 개요
* **서비스명:** 온기 터미널 (Ongi-Terminal)
* **한 줄 소개:** AI 기반 개인화 추천과 비대면 공유 거점을 결합해, 버려질 물건의 재사용과 재활용을 돕는 지역 자원 순환 플랫폼
* **서비스 형태:** 모바일 중심 반응형 웹 (PWA 적용 고려, QR 기반 비대면 인증 시스템)

## 2. 문제 정의 (Problem)
* **기존 무료나눔 플랫폼의 한계:** 채팅 기반 거래로 인한 시간 조율 피로, 노쇼, 개인정보 노출, 소액 물품 거래의 번거로움으로 인한 폐기 증가.
* **재활용 참여 문제:** 복잡한 분리배출 기준과 즉각적인 보상 부족으로 인한 실천 동기 약화.

## 3. 해결 방안 (Solution)
1. **비대면 익명 공유:** QR 인증을 통한 채팅 없는 물건 보관 및 수령으로 거래 피로도 및 분쟁 최소화.
2. **AI 기반 순환 추천:** 사용자의 관심사 및 물품 정보를 분석하여 개인화된 물품 추천 및 재사용 가능성 극대화.
3. **환경 행동의 습관화:** 재활용 반납 및 나눔 활동에 따른 '온기 지수'와 포인트 보상 체계 구축.

## 4. 핵심 타겟
* **Primary:** 20~30대 1~2인 가구, 중고거래 번거로움을 피하고 싶은 사용자.
* **Secondary:** 지역 커뮤니티 및 앱테크/친환경 활동에 관심 있는 사용자.

## 5. 핵심 기능
* **Track A (비대면 무상 공유):** AI 자동 분류/태깅이 포함된 물품 등록(Check-in) 및 주변 터미널 QR 수령(Check-out).
* **Track B (재활용 참여):** 재활용 품목 반납, AI 재활용 코칭, 포인트 및 온기 지수 획득.

## 6. 서비스 차별성
| 구분 | 기존 플랫폼 | 온기 터미널 |
| :--- | :--- | :--- |
| **거래 방식** | 채팅 기반 | 비대면 QR 기반 |
| **개인정보** | 노출 가능성 있음 | 익명성 강화 |
| **거래 구조** | 거래 중심 | 지역 자원 순환 중심 |

## 7. MVP 개발 범위
* 반응형 웹, QR 생성/인증, AI 추천 시스템, AI 챗봇, 지도 기반 UI, 포인트 및 온기 지수 시스템.

## 8. DB 스키마 (SQL)

```sql
-- 1. 사용자 정보
CREATE TABLE users (
    id SERIAL PRIMARY KEY,
    email VARCHAR(255) UNIQUE NOT NULL,
    nickname VARCHAR(100) NOT NULL,
    password_hash VARCHAR(255) NOT NULL,
    role VARCHAR(50) DEFAULT 'user',
    warmth_score INT DEFAULT 0,
    point_balance INT DEFAULT 0,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 2. 사용자 관심사
CREATE TABLE user_interests (
    id SERIAL PRIMARY KEY,
    user_id INT REFERENCES users(id),
    tag_name VARCHAR(50) NOT NULL
);

-- 3. 온기 터미널 위치
CREATE TABLE terminals (
    id SERIAL PRIMARY KEY,
    name VARCHAR(100) NOT NULL,
    address TEXT NOT NULL,
    latitude DECIMAL(10, 8),
    longitude DECIMAL(11, 8),
    status VARCHAR(50) DEFAULT 'active',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 4. 나눔 물품 정보
CREATE TABLE items (
    id SERIAL PRIMARY KEY,
    donor_id INT REFERENCES users(id),
    terminal_id INT REFERENCES terminals(id),
    title VARCHAR(255) NOT NULL,
    description TEXT,
    category VARCHAR(100),
    image_url TEXT,
    status VARCHAR(50) DEFAULT 'registered' -- registered, stored, reserved, taken, expired, rejected
);

-- 5. QR 인증용 토큰
CREATE TABLE qr_tokens (
    id SERIAL PRIMARY KEY,
    token VARCHAR(255) UNIQUE NOT NULL,
    user_id INT REFERENCES users(id),
    item_id INT REFERENCES items(id),
    terminal_id INT REFERENCES terminals(id),
    qr_type VARCHAR(50), -- item_checkin, item_checkout, recycle_checkin
    expired_at TIMESTAMP,
    used_at TIMESTAMP,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 6. 물품 이동 기록
CREATE TABLE item_transactions (
    id SERIAL PRIMARY KEY,
    item_id INT REFERENCES items(id),
    donor_id INT REFERENCES users(id),
    receiver_id INT REFERENCES users(id),
    terminal_id INT REFERENCES terminals(id),
    action_type VARCHAR(50), -- registered, checked_in, checked_out, received
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 7. 재활용 인증 기록
CREATE TABLE recycle_records (
    id SERIAL PRIMARY KEY,
    user_id INT REFERENCES users(id),
    terminal_id INT REFERENCES terminals(id),
    recycle_type VARCHAR(50),
    quantity INT,
    image_url TEXT,
    ai_result TEXT,
    point_earned INT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 8. 포인트 관리
CREATE TABLE point_transactions (
    id SERIAL PRIMARY KEY,
    user_id INT REFERENCES users(id),
    amount INT NOT NULL,
    transaction_type VARCHAR(50), -- earn, spend, adjust
    reason VARCHAR(255),
    related_id INT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 9. 온기 메시지
CREATE TABLE warmth_messages (
    id SERIAL PRIMARY KEY,
    item_id INT REFERENCES items(id),
    sender_id INT REFERENCES users(id),
    receiver_id INT REFERENCES users(id),
    message TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 10. 리워드 상점 및 교환 내역
CREATE TABLE rewards (
    id SERIAL PRIMARY KEY,
    name VARCHAR(255) NOT NULL,
    description TEXT,
    required_points INT NOT NULL,
    stock INT DEFAULT 0,
    image_url TEXT
);

CREATE TABLE reward_exchanges (
    id SERIAL PRIMARY KEY,
    user_id INT REFERENCES users(id),
    reward_id INT REFERENCES rewards(id),
    points_used INT NOT NULL,
    status VARCHAR(50) DEFAULT 'pending',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
